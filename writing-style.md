In all produced artifacts — code comments, commit messages, PR bodies, GH comments, docs — use plain, natural English. Avoid AI-tell phrasings and invented neologisms. Concrete example flagged by the user: do NOT write 'red-ing' / 'reds the shard' — write 'fails' or 'causes the shard to fail'. Prefer the ordinary word over a coined one. Also: never reference internal crosslink issue numbers in external artifacts (they resolve to unrelated GitHub issues) — cite GitHub issue numbers only.
