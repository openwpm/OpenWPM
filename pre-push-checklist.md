Mandatory checklist to complete before every `git push` or `gh pr create`.

## Checklist

### 1. Run Pre-Commit Hooks
```bash
pre-commit run --all-files
```
Fix any failures before proceeding. Do not use `--no-verify` to bypass hooks.

### 2. No AI Attribution
Never add any of the following to commits, PRs, or code:
- `Co-Authored-By: Claude` or any AI attribution lines
- `Generated by Claude`, `Generated with Claude Code`, or similar
- Any Anthropic or AI tool attribution

### 3. Strip Internal References
Run the strip-references procedure (see knowledge page `strip-references`) to remove all internal design-doc references (AC-N, REQ-N, CL-N) and VDD jargon from commits, PR body, and code.

### 4. Verify Push Target
Ensure the branch pushes to its own remote ref, not to a tracking branch that belongs to another workflow:
```bash
git rev-parse --abbrev-ref --symbolic-full-name @{u} 2>/dev/null
```
The upstream should match `origin/<your-branch-name>`. If it tracks a different branch, set it explicitly:
```bash
git push -u origin HEAD
```

### 5. Final Diff Review
```bash
git diff master...HEAD --stat
git log master..HEAD --oneline
```
Confirm no unintended files, debug code, or temporary changes are included.
